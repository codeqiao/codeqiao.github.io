指令

指令集与计算机的性能

CPUtime三要素：

  • CPI : 程序执行的指令数

  • IC : 处理一条指令所需要的始终周期数

  • T: 一个时钟周期所需要的时间

  • 三要素是相关的

  • 对于一个比值而言，不能使用算数平均值，得用几何均值才有意义

寻址方式

  • 寄存器寻址
    • Add R4, R3
  • 立即数寻址
    • Add R4, #3
  • 偏移寻址
    • Add R4, 100(R1)
    • Mem[100 + Regs[R1]]
  • 寄存器间接寻址
    • Add R4, (R1)
    • Mem[Regs[R1]]
  • 基址变址寻址
    • Add R3, (R1 + R2)
    • Mem[Regs[R1] + Regs[R2]]
  • 直接寻址
    • Add R1, (1001)
    • Mem[1001]
  • 存储器间接寻址
    • Add R1, @(R3)
    • Mem[Mem[Regs[R3]]]
  • 。。。

指令集的发展和改进

  • 复杂指令集CISC
  • 精简指令集RISC

CISC指令集功能设计

  • 面向目标程序增强指令功能
  • 面向高级语言和编译程序改进指令系统
  • 面向操作系统的优化实现改进指令系统

CISC指令集的缺点

  • 指令系统的复杂性导致了硬件系统的复杂性，以及电量的高消耗
  • 80%的指令只在20%的时间内被使用

RISC指令集的设计

  • 每条指令的功能尽可能简单，并且在一个机器周期内完成。
  • 所有指令长度均相同
  • 只有load和store指令才可以访问存储器，其他指令操作均在寄存器之间进行

MIPS指令系统解构

load/store型指令集结构

  • 具有一个简单的load/store指令集
  • 注重指令流水效率
  • 简化指令的译码
  • 高效支持编辑器

指令格式

I类型指令

操作码 rs1 rd 立即数

R类型指令

操作码 rs1 rs2 rd Func

J类型指令

操作码 与PC相加的偏移量
